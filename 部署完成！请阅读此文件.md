## 📋 部署前必须完成的准备

### ✅ 阿里云 ECS 准备

1. **创建 ECS 实例**
   - 建议配置：2 核 CPU / 2GB 内存
   - 操作系统：CentOS 7+ 或 Ubuntu 18.04+

2. **安装 Docker**（如果未安装）
   ```bash
   curl -fsSL https://get.docker.com | bash -s docker --mirror Aliyun
   systemctl start docker && systemctl enable docker
   ```

3. **安装 Docker Compose**
   ```bash
   curl -L "https://github.com/docker/compose/releases/download/v2.20.0/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
   chmod +x /usr/local/bin/docker-compose
   ```

4. **配置安全组（重要！）**
   - 登录阿里云控制台
   - ECS → 安全组 → 配置规则
   - 添加入站规则：端口 80，授权对象 0.0.0.0/0

### ✅ 本地环境准备

1. **确认可以 SSH 登录 ECS**
   ```bash
   ssh root@你的 ECS-IP
   ```

2. **确认已安装 Git、SSH、SCP**
   ```bash
   git --version
   ssh -V
   scp -V
   ```

## ⚙️ 关键技术点说明

### 1. 前后端通信架构

```
浏览器
  ↓
Nginx (80 端口)
  ├→ 前端静态文件 (/)
  └→ API 代理 (/api/* → backend:3001)
       ↓
  后端服务 (3001 端口)
```

### 2. 关键配置

**前端环境变量** (`.env`):
```env
VITE_API_URL=/api
```
- 使用相对路径，通过 Nginx 代理访问后端
- 避免硬编码 IP 地址

**Nginx 代理配置** (`nginx.conf`):
```nginx
location /api/ {
  proxy_pass http://backend:3001;
  proxy_buffering off;  # SSE 流式传输需要
}
```

**Docker 网络** (`docker-compose.yml`):
```yaml
networks:
  app-network:
  driver: bridge
```
- 三个容器在同一网络
- 通过容器名通信

## ⚠️ 常见问题

### Q1: 无法访问网页
**解决**: 
1. 检查阿里云安全组是否开放 80 端口
2. `systemctl status firewalld`
3. `docker-compose logs nginx`

### Q2: API 请求失败（CORS 错误）
**解决**:
1. 检查 `.env`: `VITE_API_URL=/api`
2. 重新构建前端：`docker-compose build frontend`
3. 清除浏览器缓存

### Q3: 构建速度慢
**解决**:
- 首次构建较慢是正常的
- 配置 Docker 镜像加速
- 后续构建会使用缓存

### Q4: 容器无法启动
**解决**:
```bash
docker-compose logs
netstat -tulpn | grep :80
df -h
```
